# Refactor Plan: Modular ESP32 Shutter Controller

## Goal

Split the monolithic `src/main.cpp` into focused, single-responsibility modules so that adding new actuators (motors, buttons, etc.) requires minimal changes to existing code.

## Proposed File Structure

```
src/
  main.cpp              ŌåÆ slim entry point: setup() / loop() only
  config.h              ŌåÆ all pin numbers, defaults, timeouts, NVS keys
  shutter.h / .cpp      ŌåÆ Shutter class (servo, angles, open/close, snapshot)
  settings.h / .cpp     ŌåÆ generic NVS read/write helper
  web.h / .cpp          ŌåÆ WebServer registration + route handlers
  wifi_manager.h / .cpp ŌåÆ WiFi connect + auto-reconnect
  html.h                ŌåÆ embedded HTML string, isolated from logic
include/
  wifi_secrets.h        ŌåÆ (unchanged)
```

## Module Details

### 1. `config.h` ŌĆö Central Configuration

All magic numbers and pin assignments in one place. No logic, just constants.

```cpp
#pragma once
#include <Arduino.h>

namespace config {
    // Pins
    static const int SHUTTER_SERVO_PIN = 27;

    // Defaults
    static const int SHUTTER_OPEN_ANGLE  = 106;
    static const int SHUTTER_CLOSE_ANGLE = 74;

    // Servo
    static const int SERVO_MIN_PULSE = 500;
    static const int SERVO_MAX_PULSE = 2400;
    static const int SERVO_HZ        = 50;

    // NVS
    static const char* NVS_NAMESPACE     = "shutter";
    static const char* NVS_KEY_OPEN      = "openAngle";
    static const char* NVS_KEY_CLOSE     = "closeAngle";

    // Network
    static const uint16_t WEB_PORT = 80;
}
```

When new motors/buttons are added, their pins and defaults go here.

### 2. `settings.h / .cpp` ŌĆö NVS Persistence

A thin wrapper around `Preferences`. Two functions: `loadInt` / `saveInt`.

```cpp
// settings.h
#pragma once
#include <Arduino.h>

namespace settings {
    int  loadInt(const char* ns, const char* key, int fallback);
    void saveInt(const char* ns, const char* key, int value);
}
```

Implementation opens prefs, reads/writes, closes. No global state.

### 3. `shutter.h / .cpp` ŌĆö Shutter Class

Encapsulates one shutter: servo object, open/close angles, current state, snapshot timer.

```cpp
// shutter.h
#pragma once
#include <Arduino.h>
#include <ESP32Servo.h>

class Shutter {
public:
    void begin(int pin, int openAngle, int closeAngle);
    void open();
    void close();
    void snapshot(unsigned long durationMs);
    void loop();  // call from main loop ŌĆö handles auto-close

    int   getAngle()   const;
    bool  isOpen()     const;
    int   openAngle()  const;
    int   closeAngle() const;

    void setOpenAngle(int angle);
    void setCloseAngle(int angle);

private:
    Servo          _servo;
    int            _pin = -1;
    int            _openAngle  = 106;
    int            _closeAngle = 74;
    bool           _isOpen = false;
    unsigned long  _autoCloseAt = 0;
};
```

Key points:
- `begin()` attaches the servo and moves to closed position.
- `loop()` checks `_autoCloseAt` for non-blocking snapshot auto-close.
- `snapshot()` opens then sets `_autoCloseAt`.
- Adding a second actuator (e.g., focus motor) means creating a different class with a similar shape ŌĆö not duplicating code in `main.cpp`.

### 4. `wifi_manager.h / .cpp` ŌĆö WiFi Connection

```cpp
// wifi_manager.h
#pragma once
#include <Arduino.h>

namespace wifi {
    void connect(const char* ssid, const char* pass);
    bool isConnected();
    void reconnectIfNeeded();  // call from loop()
}
```

Keeps WiFi connection logic out of `main.cpp`. `reconnectIfNeeded()` can be called every loop iteration for resilience.

### 5. `html.h` ŌĆö Embedded HTML Page

The raw string literal moves here as a `constexpr` or `PROGMEM` string. The `%d` / `%s` placeholders remain ŌĆö the web module does the `snprintf` formatting. This is the only file you touch when changing the UI.

### 6. `web.h / .cpp` ŌĆö Web Interface

Owns the `WebServer` instance and all route handlers. Takes a reference to the `Shutter` object.

```cpp
// web.h
#pragma once
#include <WebServer.h>
#include "shutter.h"

class WebInterface {
public:
    void begin(Shutter& shutter);
    void loop();

private:
    void handleRoot();
    void handleOpen();
    void handleClose();
    void handleSave();
    void handleSnapshot();

    WebServer _server{80};
    Shutter*  _shutter = nullptr;
};
```

Route handlers call `_shutter->open()`, `_shutter->close()`, etc. Adding new endpoints (e.g., for a focus motor) means adding methods here.

### 7. `main.cpp` ŌĆö Slim Entry Point

```cpp
#include <Arduino.h>
#include "config.h"
#include "shutter.h"
#include "web.h"
#include "wifi_manager.h"
#include "wifi_secrets.h"

static Shutter       shutter;
static WebInterface  web;

void setup() {
    Serial.begin(115200);
    delay(500);

    int openAngle  = settings::loadInt(config::NVS_NAMESPACE, config::NVS_KEY_OPEN,  config::SHUTTER_OPEN_ANGLE);
    int closeAngle = settings::loadInt(config::NVS_NAMESPACE, config::NVS_KEY_CLOSE, config::SHUTTER_CLOSE_ANGLE);

    shutter.begin(config::SHUTTER_SERVO_PIN, openAngle, closeAngle);
    wifi::connect(WIFI_SSID, WIFI_PASS);
    web.begin(shutter);
}

void loop() {
    wifi::reconnectIfNeeded();
    shutter.loop();
    web.loop();
}
```

## Execution Order

1. Create `src/config.h` ŌĆö extract all constants
2. Create `src/settings.h` + `src/settings.cpp` ŌĆö NVS helper
3. Create `src/shutter.h` + `src/shutter.cpp` ŌĆö Shutter class
4. Create `src/html.h` ŌĆö move HTML string
5. Create `src/web.h` + `src/web.cpp` ŌĆö WebInterface class
6. Create `src/wifi_manager.h` + `src/wifi_manager.cpp` ŌĆö WiFi helper
7. Rewrite `src/main.cpp` ŌĆö slim entry point
8. Build and verify with `pio run`

Each step leaves the project in a compilable state ŌĆö we can build after every 1-2 steps.

## What This Enables

- **New motor** (e.g., focus motor): create a `FocusMotor` class, add pin to `config.h`, instantiate in `main.cpp`, add endpoints to `WebInterface`.
- **Physical button**: add a `Button` class or simple polling in `main.cpp`, call `shutter.open()` / `shutter.close()`.
- **UI changes**: only edit `html.h`.
- **WiFi changes**: only edit `wifi_manager.cpp`.
